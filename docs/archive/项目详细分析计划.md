# 项目详细分析计划

> 本文档规划了对新闻分析系统所有核心文件的审阅和文档编写计划，确保完整覆盖工作流运转逻辑和兜底措施。

## 一、项目概况

- **项目名称**: 新闻分析系统 (news_analyzer)
- **核心功能**: RSS多源采集、AI智能过滤、深度报告生成
- **技术栈**: Python + SQLite + ChromaDB + RAG
- **分析目标**: 编写完整的工作流运转逻辑说明文档（面向后端开发/测试）

## 五、关键关注点

### 5.1 必须详细说明的兜底机制

1. **RSS采集兜底**: 官方 → RSSHub → Google News → 第三方
2. **AI批处理兜底**: 失败时标记为"待分类"，后续重判
3. **数据库写入兜底**: 指数退避重试(最多3次)
4. **遗漏补救机制**: 检测 → 扩大回溯 → 验证效果
5. **任务锁兜底**: 超时自动清理失效锁

### 5.2 必须详细说明的工作流

1. **Task1采集6阶段**: 采集 → 过滤 → AI校验 → 入库 → 兜底重判 → 备份
2. **Task2报告7阶段**: 加载 → 去重 → 简报 → 深度报告 → TOP N选择 → 邮件 → 归档
3. **增量采集策略**: 智能回溯计算、RSS滚动边界检测
4. **五维评分体系**: 权重计算、字段映射

### 5.3 必须详细说明的数据流转

1. RSS Item → News Dict → NewsData → Database
2. 待分析池 → 归档池的流转
3. 知识库向量化的触发时机
